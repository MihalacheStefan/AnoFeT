AnoFeT

Abstract

AnoFeT (prescurtarea de la Anonymous Feedback Tool) este un instrument Web ce permite utilizatorilor posibilitatea de a oferi feedback pentru un anumit "lucru" intr-o maniera anonima.

Introduction

Documentatia doreste sa prezinte o problema contemporana, oferirea unui feedback legat de un anumit "lucru", deoarece aceste feedback-urile influenteaza foarte mult luare deciziilor de catre indivizi. Utilizatorii instrumentului Web vor putea creea un anumit Topic, specific preferintelor lor, vor putea oferi feedback altor lucruri si vor putea consultat Topul si Statistica tuturor lucrurilor.

Structure

Din punct de vedere al structurii, am ales sa folosim design pattern-ul MVC, un model arhitectural ce imparte aplicatia in trei parti interconectate. Cele trei parti sunt Model-ul responsabil cu comunicarea dintre aplicatie si baza de date, View-ul responsabil cu afisarea informatiilor catre client si Controller-ul responsabil cu logica aplicatiei si cu asigurarea comunicarii dintre Model si View.

Structura folosita nu foloseste API-uri externe, informatiile fiind stocate si prelucrate in cadrul bazei de date si al serverului.

Baza de date este formata din 6 tabele : accounts, object_user, objects, opinions, questionnaire, answers ce au rolul de a gestiona datele introduse de catre client dupa cum urmeaza.

Semantics Overlay

Din punct de vedere al limbajelor utilizate, instrumentul Web utilizeaza comenzi de tip PHP dar si script-uri, HTML, si CSS.

Persons & Organizations

Organizarea in cadrul echipei s-a realizat in baza prezentarilor efectuate. Astfel sarcinile au fost distribuite in functie de Componenta Front-end, Componenta Arhitecturala si Componenta Finala.

Conponenta Front-end a presupus elaborarea UI-ului, prin scrierea de cod HTML si CSS specifice paginilor ce vor fi ulterior modelate in View.

Conponenta Arhitecturala a presupus elaborarea design-ului arhitectural al instrumentului Web (Design Patterns, Diagrama de clase, Diagrama Master, Structura bazei de date, Usecase-uri).

Conponenta Finala a presupus implementarea efectiva a instrumentului Web, limbajul principal fiind PHP.

Pentru mai multe detalii referitoare la dezvoltarea proiectului si la contributia fiecarui membru al echipei, recomandam consultarea paginii de GitHub aferenta proiectului.

Implementing Details

Baza de date aleasa este MySql, codul fiind scris in concordanta cu design pattern-ul ales; astfel codul aferent bazei de date a fost inserat in fisierele din cadrul Modelului, in functii denumite corespunzator. Fiecare functie creeaza o conexiune cu baza de date, executa interogarea sql dupa care inchide conexiunea si returneaza rezultatul interogarii. Spre exemplu functia insert_obj_user ce primeste ca parametrii Id-ul obiectului, Id-ul user-ului si numarul de ore active ale chestionarului, care creeaza o conexiune cu baza de date, pregateste datele ce vor fi inserate si executa query-ul SQL.

Apoi, la nivelul fisierelor din Controller, exista atat fisiere cu functii care doar ofera comunicarea dintre Model si View, dar si functii ce sunt responsabile de logica aplicatiei si validarea datelor.

La nivelul fisierelor din View, acestea sunt compuse in principal din cod PHP si HTML, avand rolul de a oferi informatia clientului.
